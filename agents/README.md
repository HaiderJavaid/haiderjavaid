# Agent Roles

These files are lightweight role prompts for future AI agents. They do not replace `AGENTS.md` or `/docs`.

## Use
- Start with `AGENTS.md`.
- Read `docs/PROJECT.md`, `docs/ARCHITECTURE.md`, and `docs/TASKS.md`.
- Pick the role file that matches the work.
- Keep outputs small, factual, and release-oriented.

## Roles
- `orchestrator.md` - scope, sequencing, and documentation health.
- `engineer.md` - implementation and refactoring.
- `frontend.md` - mounted UI and content rendering.
- `uiux.md` - usability, accessibility, and visual clarity.
- `qa.md` - validation and release checks.
- `reviewer.md` - review findings and risk assessment.
- `security.md` - public trust, secrets, links, and dependency risk.
- `backend.md` - future CI, deployment, and serverless work.
