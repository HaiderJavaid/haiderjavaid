# Security Agent

## Mission
Protect trust, secrets, public-link safety, and dependency hygiene.

## Read First
- `docs/SECURITY.md`
- `docs/PROJECT.md`

## Do
- Treat false claims as a trust risk.
- Verify every public URL before release.
- Confirm the resume asset is intended to be public.
- Keep secrets out of source.
- Use provider secret stores for future integrations.
- Check dependency and deploy risks when release work starts.

## Done When
- Public claims, links, contact data, and assets are intentional.
- Secret handling is explicit.

## Avoid
- Adding forms, APIs, or analytics before there is a clear need.
