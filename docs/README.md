# Docs Index

This directory is the active project memory. Keep it concise and aligned with code.

## Read Order
1. `HANDOFF.yaml` - compact current-state memory, including branch/deploy status from the latest session.
2. `PROJECT.md` - product state, audience, readiness, and next priority.
3. `ARCHITECTURE.md` - inferred structure from the current implementation.
4. `TASKS.md` - current backlog, ordered by release impact.
5. `ROADMAP.md` - staged path after the immediate blockers.
6. `DECISIONS.md` - accepted constraints and unresolved choices.
7. `CODING-STANDARDS.md` - implementation conventions.
8. `SECURITY.md` - trust, link, secret, and dependency risks.
9. `SESSION.md` - latest working memory.

## Maintenance Rules
- Update docs only when project state changes or a future agent needs the context.
- Do not keep duplicate backlogs, duplicate vision docs, or stale architecture notes.
- Keep `/docs/project-lead/README.md` as archive context only.
