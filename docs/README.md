# Docs Index

This directory is the active project memory. Keep it concise and aligned with code.

## Read Order
1. `PROJECT.md` - product state, audience, readiness, and next priority.
2. `ARCHITECTURE.md` - inferred structure from the current implementation.
3. `TASKS.md` - current backlog, ordered by release impact.
4. `ROADMAP.md` - staged path after the immediate blockers.
5. `DECISIONS.md` - accepted constraints and unresolved choices.
6. `CODING-STANDARDS.md` - implementation conventions.
7. `SECURITY.md` - trust, link, secret, and dependency risks.
8. `SESSION.md` - latest working memory.

## Maintenance Rules
- Update docs only when project state changes or a future agent needs the context.
- Do not keep duplicate backlogs, duplicate vision docs, or stale architecture notes.
- Keep `/docs/project-lead/README.md` as archive context only.
